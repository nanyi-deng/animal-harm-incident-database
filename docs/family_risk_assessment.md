# 家属/涉事人风险评估（HRL-002）

## 1. 目的与范围

本评估回答一个 `docs/privacy_qa_checklist.md` 不回答的问题。隐私 QA 清单问的是"数据本身有没有技术性泄漏"（有没有不该出现的姓名/住址）；本评估问的是**即使没有技术性泄漏，把这批信息集中做成一个可搜索、结构化、长期存续的公开数据库，会不会让数据里出现的真实的人处于比"信息本来就零散地存在于几篇新闻报道里"更高的风险中**——这是信息聚合本身带来的风险，不是某个字段填错了。

范围覆盖 v0.1 快照的全部 30 条事件，重点是数据集正式对外发布（Zenodo Public / 网站取消 noindex）之前必须想清楚的问题，不是发布之后才补的手续。

## 3. 已有的缓解机制（不是本次新做的，是已经落地的）

- **HRL-015 身份政策**：跟随信源本身识别程度，AHID 不主动深挖补全任何人的真名；未成年人无例外匿名。
- **HRL-008 隐私 QA**：185 段自由文本逐一扫描确认无未经批准的姓名泄漏；#21/#22 两条未成年人事件的小区级地点已模糊到市/县级。
- **AF 状态机制**：#26 大熊猫谣言案证明了流水线有能力识别并标记"已被证实为虚假指控"的内容，转成纠错记录而非继续传播指控——这是防止 AHID 自己变成二次伤害源头的核心设计。
- **申诉/更正机制**（`docs/appeals_process.md`）：高风险申诉（涉及身份、未成年人、法律威胁）14 天内处理承诺。
- **网站层 noindex/nofollow**：当前站点不被搜索引擎收录，不主动扩大曝光面。
- **PRD §1.3 定位声明**：AHID 明确定位为"不是黑名单、不是民间定罪平台"，页面本身会展示这段免责声明。

## 4. 残余风险——诚实分析，不因为已有缓解机制就假设风险为零

**风险一：信息聚合放大效应（适用于 3 条已用实名的成年人事件）**。原始新闻报道本身用了实名，这是既成事实，AHID 收录不是"泄漏"。但把这类信息从"分散在几篇可能会被时间冲淡的新闻报道里"变成"在一个结构化、可筛选、长期维护、面向研究者持续引用的数据库里"，客观上提高了这条记录未来被检索到、被反复引用的概率和持续时间——哪怕原始新闻已经被删除或不再被搜索引擎收录，AHID 也会让它继续可查。这是信息聚合本身固有的、无法靠"跟随信源"这条规则消除的风险，只能承认并想清楚是否可接受。

**风险二："旺旺事件"的特殊历史**。`methodology.md` 的 Positionality 一节已经承认：这一起案例本身曾发生网民人肉 4 名涉案未成年人的行为，项目文档明确把这定性为"争议行为，非范例"。AHID 收录这一事件本身（哪怕零身份信息）是否会成为该案例重新被提起、重新引发关注（进而可能重新触发人肉行为，即使目标不是 AHID 本身）的一个新入口，需要认真权衡——这起案例同时也是项目的起源和最有公众认知度的收录事件，完全不收录也不现实，但需要想清楚怎么呈现。

**风险三：机构声誉/法律风险**。涉事学校/公司/政府部门是机构不是自然人，人身安全风险不适用，但存在被认为"污名化"的申诉甚至法律风险，尤其是 #4、#14 这类存在矛盾陈述或已证实为虚假举报的事件——AX/AF 状态和矛盾记录如实保留正是为了避免这个问题，但机构一旦发现自己出现在一个叫"动物伤害数据库"的网站上，即使记录本身如实中立，仍可能主动提出异议。申诉机制需要真正可用（对应 HRL-019 尚未定的联系邮箱）。

**风险四：一旦通过 Zenodo/DOI 公开，不可完全撤回**。Zenodo 记录发布后不能删除内容，只能发新版本。这意味着"申诉后更正"在 Zenodo 层面实际操作是"发一个修正过的新版本"，旧版本仍然可以被找到——这跟申诃机制承诺的"更正"存在真实的操作张力，需要在申诉页面/文档里说清楚，不能让人误以为"申诉成功=旧记录消失"。

## 5. 建议的额外缓解措施

- [ ] **HRL-019 尽快定下来**：申诉联系邮箱在真正公开前必须是可用的，风险三、风险四都依赖这个机制真实存在。
- [ ] **"旺旺事件"呈现方式复核**：建议你亲自再读一遍 #1 这条记录在网站/数据集里的最终呈现文字，确认没有任何可能被解读为"猎奇""证据链"的表述，纯粹呈现事实与制度回应。
- [ ] **Zenodo 记录的"不可撤回"性质需要在文档里明确写出**：建议在 `docs/appeals_process.md` 或 `known_limitations.md` 补一句话，说明"更正"在已发布版本层面的实际含义，管理期望。
- [ ] **首次公开建议范围克制**：与其一次性把 30 条全部推向最大曝光（比如同时发 Zenodo + 取消网站 noindex + 主动推广），可以考虑先只做 Zenodo（学术引用场景，曝光面相对可控），网站取消 noindex 单独作为一个后续决定，两者不必绑在一起同时发生。
