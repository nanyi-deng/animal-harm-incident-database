# 判决书 Census 抽样协议（v0.2 子语料）——草案 v2

状态：**待批准**（HRL-023）。批准后按本协议执行；执行中的任何偏离须记回本文档。

> **v2 修订（2026-07-20）**：应用户"不想投入两天手动检索"的要求，主路径改为 **CAIL2018 离线语料库全量检索**（用户时间≈0，AI 承担全部检索与初筛），原文书网手动检索降级为可选的近年补充。v2 已实际执行完检索步骤，原始命中数见 §3。

## 1. 目的与定位

给 AHID 增加一个**有明确分母的系统性子语料**：中国裁判文书网上公开可得的、涉动物伤害的刑事判决。它与现有采集渠道（Tier D 人工回填、公众提交）**并行共存,不替换**——incident 层新增 `acquisition_channel` 字段区分：

| 取值 | 含义 |
|---|---|
| `curated_backfill` | 现有 30 条（Tier D 人工线索回填） |
| `public_submission` | 公众提交渠道审核通过的事件 |
| `judgment_census` | 本协议采集的判决书语料 |

分析时各取所需：推断性统计只用有分母的 `judgment_census`；监测与个案研究用全库。两个语料的**对比**本身是研究问题（哪类事件能进入司法程序——中国无动物保护法,判决书只覆盖严重到够上现有罪名的极小一角,公众提交恰好捕捉司法看不见的部分）。

## 2. 数据源与分工（v2：离线语料为主）

**主源：CAIL2018**（Chinese AI and Law Challenge 2018 语料库）——最高人民法院信息中心参与、"法研杯"官方发布的研究数据集，268 万份刑事判决（源自裁判文书网，单被告人案件，2018 年及以前），含事实描述全文 + 罪名标注 + 刑罚信息。出处正当（官方为研究目的发布，论文 arXiv:1807.02478），Hugging Face 可直接下载（`china-ai-law-challenge/cail2018`）。**全部检索与初筛由 AI 在本地完成，用户零手动检索时间。**

主源的三个如实记录的代价：
1. **无案号/法院/日期/地域元数据**（竞赛语料匿名化的一部分）——溯源方式改为"CAIL2018 数据集 + 记录索引"（数据集本身可引用、可核查）；对最终收录的少量案件，可用事实描述中的独特字串回文书网定位原判决（可选、逐案、仅在需要锚定时做）。
2. **时间窗止于 2018**——2019 年后的判决不在其中。近年补充为可选项（见下）。
3. **仅单被告人案件**（CAIL2018 构建时的过滤）——写入方法声明。
附带的好处：语料自带"张某某"式姓名遮蔽，与本项目身份政策（§8）天然一致。

**可选补充源（降级，不阻塞）**：裁判文书网手动检索 2019–2026 窗口（用户本人账号，缩减检索式后约半天）；北大法宝浏览器检索作覆盖面交叉核对。两站均不做自动化抓取（探明：文书网 API 未登录 code 9 拒绝；pkulaw 对程序访问 WAF 拦截 HTTP 567、真人浏览器可用——本设计下无影响）。

## 3. 检索协议（v2 已执行，2026-07-20）

**分层检索式**（在 CAIL2018 全量 fact 文本上执行）：

- **A 层（案由 × 动物 × 伤害）**：案由 ∈ {寻衅滋事, 投放危险物质, 以危险方法危害公共安全, 故意毁坏财物} 且 fact 含动物词 {犬, 狗, 猫, 动物, 宠物} 且含伤害词 {毒, 打死, 殴打, 虐, 砍, 烧, 摔, 杀害, 勒死, 掐死}
- **B 层（强搭配词，不限案由）**：fact 含 {偷狗, 毒狗, 毒镖, 虐待动物, 虐狗, 虐猫} ——捕捉以盗窃/抢劫/食品安全等罪名起诉的偷狗毒狗产业链案件，这类案件 A 层的案由过滤会漏掉

**执行结果（原始命中，未经人工筛选）**：

| 项 | 数量 |
|---|---|
| 扫描总行数（跨分片按 fact 去重后计入） | 2,168,025 |
| 目标案由行数 | 63,881 |
| A 层命中 | 368 |
| B 层新增命中（A 层未覆盖） | 386 |
| **原始候选合计** | **754** |

B 层罪名构成（前几位）：盗窃 168、抢劫 76、抢劫+盗窃 27、故意伤害 25、生产销售有毒有害食品 18、非法拘禁 10、过失致人死亡 8——证实"偷狗毒狗产业链"是判决书语料中体量可观、且完全不经动物类罪名起诉的独立现象。

**已知假阳性来源**（小样本试跑确认，精筛阶段处理）：地名含"狗"（狗湾村）、詈语中的"狗"、"石狮"类文物。

**范围决定（须在批准时确认）**：野生动物类罪名（危害珍贵濒危野生动物罪、非法狩猎罪）**暂不纳入**——盗猎/走私是另一个庞大且性质不同的语料；仅当行为符合 PRD §7.2 蓄意伤害/虐待类型时经关键词路径自然进入。**新增范围问题（试跑发现）**：投毒护田致邻里牲畜死亡类纠纷案（如毒玉米致 25 只山羊死亡）在语料中占比可能不小——是否纳入、还是单列一个子类别标注，批准时一并确认（建议：纳入但以 `harm_categories` 与动机字段区分，不与蓄意虐待混算）。

原始命中存于 `pipeline/cail_census_raw_hits.jsonl`（gitignored，754 条含判决事实全文）。

## 4. 筛选与纳入（v2 分工）

1. **去重**：CAIL2018 无案号，改按 fact 文本哈希去重（检索时已做）；残余近重复（同案不同切分）在精筛阶段人工识别。
2. **AI 精筛（下一步）**：AI 通读全部 754 条原始命中，逐条分类：真实动物伤害案（细分类型）/ 假阳性（地名、詈语、文物等）/ 边界案例（护田投毒类等，标注待用户决定），产出类似 `candidate_incidents_seed.md` 的候选审核表。
3. **用户审核**：按既有批量确认模式（HRL-016/017 先例）审核 AI 的分类结果与纳入建议——预计 1–2 小时，不是两天。
4. **纳入标准**：判决事实认定部分描述了符合 PRD §7.2 的动物伤害行为（不要求动物伤害是定罪核心——盗窃案中毒狗常是手段情节，照样纳入并如实标注）。
5. **排除**：动物只作为背景/财产价值争议而无伤害行为；人被动物所伤；纯野生动物贸易无虐待情节。
6. 每条筛选决定记入执行日志（PRISMA 风格：各阶段数量流水）。

## 5. 覆盖偏差的如实记录（v2）

- 抽样框的准确表述是"**CAIL2018 语料（268 万份 2018 年前公布于文书网的单被告人刑事判决）中的全部匹配案件**"——分母明确、语料冻结、完全可复现（任何人下载同一数据集跑同一检索式得到同一结果）。
- CAIL2018 构建于 2021 年文书网大规模下架潮**之前**,反而保留了部分后来被下架的判决——这是离线冻结语料相对实时网站检索的一个方法学优点,写入方法声明。
- 已知不覆盖：2019 年后的判决、多被告人案件、未上网判决（上网率本身有选择偏差）。与 methodology.md §10 的删除偏差/平台偏差小节衔接表述。

**证据类型透明拆分**（公开表元数据标记，供论文如实区分证据强度，不把三类混为一谈）：

- `outcome_documented=false`：72 条——产业链推定案，判决未明写动物最终结局，按作案手段/动机与已确认案例一致纳入
- `claim_verified=false`：5 条——下毒等指控未经判决认定为事实，采信声称本身计入
- `animal_directly_harmed=false`：4 条——偷狗现象案，动物本身未被证实受伤（受伤/死亡的是嫌疑人或围殴者）
- `recovered_after_theft=true`：4 条——盗窃既遂后动物被追回
- `perpetrator_confirmed=false`：4 条——动物伤害真实但加害人身份未坐实

即：公开表的 424 条中，绝大多数是判决实证的动物伤害；仅约 72 条为产业链模式推定纳入、5 条为未经认定的声称——论文报告时可据此分层，避免"424 条全部为判决实锤"的失真表述。

## 8. 待决的伦理问题（随本协议一并决定,HRL-023）

**判决书实名与既有身份政策的交叉**：刑事判决书对被告人常用实名(或"张某华"式部分遮蔽,各法院不一)。HRL-015 规则是"跟随一手/官方信源的识别程度"——判决书就是最权威官方信源,按规则可沿用其披露程度。但存在一种新情形:**同一事件此前新闻报道匿名、判决书披露更多**（如 #28 张某华,若判决书署全名）。按 HRL-015 字面,官方信源披露即可沿用;但这意味着 AHID 会比部分公众报道"更具名"。需要你明确:(a) 照 HRL-015 字面执行(判决书披露即沿用)——立场一致、简单;(b) 对 judgment_census 语料统一采用判决书原文的遮蔽程度,不做任何补全或还原——即照抄文书,文书写"张某华"就写"张某华",写全名就写全名。**建议 (b)**,它其实就是 (a) 的严格执行版,重点是绝不跨文书拼接身份。

## 10. 发布前验证与 stage-gate（data-pipeline-prd 规范，2026-07-21 追加）

本节按 `/data-pipeline-prd` skill 的原则回溯补写：**AI/工具的输出是一个待核验的主张，不是事实**，管线的每一个自动化阶段都需要一个可运行、可复现的不变量检查，下游只在上游 gate 通过时才消费其输出。本 census 的验证脚本是 `pipeline/census_validate.py`（结构不变量）与 `pipeline/census_privacy_scan.py`（隐私）。

### Stage-gate 表

| Gate | 阶段 / 产物 | 不变量与检查 | 状态 |
|---|---|---|---|
| G1 | 检索 → `cail_census_raw_hits.jsonl` | (file,idx) 全唯一，无键碰撞；行数=检索命中 | ✓ 754 唯一，无碰撞 |
| G2 | AI 分类（6 批）→ 类别 | 分类覆盖 754/754；抽样人工核验一致度 | ⚠ 覆盖 ✓；**核验有盲区，见下** |
| G3 | 合并 → `census_classified.jsonl` | 输入 754 = 输出 754，join 无失配、无重复 census_id | ✓ 对账通过 |
| G4 | 规则校正（override 文件）| 校正实例集须穷举其纠正的缺陷类，非仅"碰巧看到的" | ⚠ **基于 93 条样本，非全量，见下** |
| G5 | 去重 → `census_dedup_pairs.json` | 近重复识别、保留不合并、如实报告有效唯一数 | ✓ 92 对；有效 ~342/424 |
| G6 | 应用审核 + 导出 → `judgments_census.csv` | 无 fp 泄漏、fact 100% 非空、census_id 与 file/idx 自洽、证据标记与 category 对齐 | ✓ census_validate 全过 |
| G7 | 隐私 → 发布前扫描 | 零结构化 PII；遮蔽普及度；未遮蔽第三方姓名逐一分类 | ✓ 0 手机号/身份证，98% 遮蔽（HRL-024） |

### Test-before-patch 与 patch 计数

`pipeline/census_apply_corrections.py` 是一个 override 文件，按 census_id 手工纠正具体记录。当前 patch 计数：CRUELTY_RECLASSIFY 15、CRUELTY_UNVERIFIED_CLAIM 5、THEFT_PHENOMENON 5、FP↔POACHING 修正 2、其余若干。skill 规则要求 override 只有在"某阶段测试已穷举其纠正的缺陷类的全部实例"之后才可信。**本 census 未满足这一条**：校正来自 93 条人工审核集，而非对 754 条的系统枚举——因此校正只修好了恰好落进样本里的实例。

### 已知验证盲区（回溯审计 2026-07-21 发现，如实披露）

回溯审计跑了 skill 的完整性与差分检查，确认 G1/G3/G5/G6/G7 干净，但在 G2/G4 查出三处真实盲区，直接影响数据集的**完整性**（不是它保留了错的，而是它可能漏了对的）：

1. **假阳性排除未经核验（最重要）。** 329 条被 AI 判为 fp 的记录整批排除出公开表，其中仅 20 条（随机样本）经人工复核——而这 20 条里有 6 条被救回为真实案件（30%）。这些多是 AI 按"动物是否被直接伤害"的较严标准判 fp、却被用户"偷狗情节即计入"的较宽政策纳入的偷狗案。按此比例外推，329 条 fp 中约有 **~99 条真实案件可能被静默排除**——恰好落在本 census 最核心的"偷狗现象"类别上，意味着头号发现的规模被系统性低估。
2. **AI 分类无差分交叉验证。** G2 只有一次人工抽样，没有用第二种方法（二次分类、不同提示词）对同一批做差分——而差分是 AI 转换类阶段性价比最高的不变量。
3. **346 条自动纳入行携带未校正的 AI 分类。** 高置信非假阳性直接纳入、未经人工过目；G4 的校正只作用于审核集，这 346 条按当时的 AI 错误率原样进入数据集。

### 排序约束与复现节奏

- **排序约束**：`judgments_census.csv` 与任何据其计算的论文数字，在 G1–G7 全绿之前不冻结。上游的分类修正若在冻结后落地，会静默作废已据旧版算出的数字。
- **复现节奏**：三个强制检查点——(1) 人工审核批次定稿前；(2) 任何上游重跑（重检索/重分类）合并前；(3) 任何将成为论文报告数字的运行前。CAIL2018 是冻结语料，无第四检查点（模型/工具版本漂移）之虞。
- **审核集复现**：原始抽样在 `census_review_items.py` 加固定种子之前完成，实际审核集以 `pipeline/census_audit_set_ids.txt`（93 条 ID 冻结记录）为权威版本。
