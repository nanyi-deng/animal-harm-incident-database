# 判决书 Census 抽样协议（v0.2 子语料）——草案

状态：**待批准**（HRL-023）。批准后按本协议执行；执行中的任何偏离须记回本文档。

## 1. 目的与定位

给 AHID 增加一个**有明确分母的系统性子语料**：中国裁判文书网上公开可得的、涉动物伤害的刑事判决。它与现有采集渠道（Tier D 人工回填、公众提交）**并行共存,不替换**——incident 层新增 `acquisition_channel` 字段区分：

| 取值 | 含义 |
|---|---|
| `curated_backfill` | 现有 30 条（Tier D 人工线索回填） |
| `public_submission` | 公众提交渠道审核通过的事件 |
| `judgment_census` | 本协议采集的判决书语料 |

分析时各取所需：推断性统计只用有分母的 `judgment_census`；监测与个案研究用全库。两个语料的**对比**本身是研究问题（哪类事件能进入司法程序——中国无动物保护法,判决书只覆盖严重到够上现有罪名的极小一角,公众提交恰好捕捉司法看不见的部分）。

## 2. 数据源与分工

| 源 | 角色 | 访问方式 | 已探明状态（2026-07-20） |
|---|---|---|---|
| 裁判文书网 wenshu.court.gov.cn | **正本来源**（判决全文,Tier 1） | 用户本人账号,浏览器手动检索+导出 | 首页可达；检索 API 未登录拒绝（code 9）；注册需大陆手机号 |
| 北大法宝 pkulaw.com | **辅助索引**（交叉核对覆盖面/命中数） | 用户浏览器手动检索 | 真人浏览器可开；程序访问被 WAF 拦（HTTP 567 挑战页）；匿名全文权限待用户实测 |

**明确不做自动化抓取**：两站都有反爬/登录墙,绕过与 PRD §11.3 冲突且有封号风险;体量（预估几十到低百件）完全在手动范围内。手动检索按系统综述（systematic review）标准操作:预先固定检索式,记录每步命中数,可复现、可审计。

## 3. 检索协议

**时间窗**：判决日期 2014-01-01 至执行日（文书网 2014 年起上网较全）。执行日即 as-of date,写入日志。

**检索式设计**（案由 × 关键词,两库同式执行）：

案由（刑事）：
- 寻衅滋事罪
- 投放危险物质罪
- 以危险方法危害公共安全罪
- 故意毁坏财物罪（动物作为财物被蓄意伤害的路径）

全文关键词（每个案由分别组合检索,记录各查询命中数）：
- 虐待动物 / 虐狗 / 虐猫
- 毒狗 / 毒猫 / 毒鸟
- 殴打+犬 / 殴打+猫（或各库支持的邻近检索语法）
- 电击+动物（视命中情况增删,增删记录在日志）

**范围决定（须在批准时确认）**：野生动物类罪名（危害珍贵濒危野生动物罪、非法狩猎罪）**暂不纳入**——盗猎/走私是另一个庞大且性质不同的语料,会淹没本子语料;仅当行为本身符合 PRD §7.2 的蓄意伤害/虐待类型（如虐待展演野生动物）时经由关键词路径自然进入。此边界如实写入方法文档。

## 4. 筛选与纳入

1. **去重**：按案号;同一事件的一审/二审/再审 = 同一 Incident 的多个 Source（判决链完整保留）。
2. **纳入**：判决书事实认定部分描述了符合 PRD §7.2 的动物伤害行为（不要求动物伤害是定罪核心——寻衅滋事案中动物伤害常是情节之一,照样纳入并如实标注）。
3. **排除**：动物只作为背景/财产价值争议而无伤害行为;人被动物所伤;纯野生动物贸易无虐待情节。
4. 每条筛选决定记入执行日志（PRISMA 风格:各阶段数量流水）。

## 5. 删除偏差的如实记录

文书网 2021 年起大规模下架判决（刑事类尤甚）。处理:
- 抽样框的准确表述是"**截至 as-of 日文书网公开可得的全部匹配判决**",不声称是历史全量;
- 两库同式检索的命中数差值记录在案——pkulaw 收录但文书网已下架的判决,以 pkulaw 文本作为官方文书的转录副本纳入（primary_source_status = `near_primary`,availability 备注注明正本已从文书网下架）;
- 此偏差与 methodology.md §10 已有的"删除偏差"一节衔接,发布时合并表述。

## 8. 待决的伦理问题（随本协议一并决定,HRL-023）

**判决书实名与既有身份政策的交叉**：刑事判决书对被告人常用实名(或"张某华"式部分遮蔽,各法院不一)。HRL-015 规则是"跟随一手/官方信源的识别程度"——判决书就是最权威官方信源,按规则可沿用其披露程度。但存在一种新情形:**同一事件此前新闻报道匿名、判决书披露更多**（如 #28 张某华,若判决书署全名）。按 HRL-015 字面,官方信源披露即可沿用;但这意味着 AHID 会比部分公众报道"更具名"。需要你明确:(a) 照 HRL-015 字面执行(判决书披露即沿用)——立场一致、简单;(b) 对 judgment_census 语料统一采用判决书原文的遮蔽程度,不做任何补全或还原——即照抄文书,文书写"张某华"就写"张某华",写全名就写全名。**建议 (b)**,它其实就是 (a) 的严格执行版,重点是绝不跨文书拼接身份。
