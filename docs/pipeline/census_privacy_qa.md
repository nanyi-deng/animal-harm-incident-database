# 判决书 Census 隐私 QA（发布前）

配套脚本：`pipeline/census_privacy_scan.py`。对应事件库的 `docs/privacy_qa_checklist.md` + `pipeline/privacy_scan.py`，但针对 census 的特殊风险：**公开表 `judgments_census.csv` 包含 424 条判决的事实认定全文**，发布前提是"CAIL2018 已遮蔽人名"。本文核实这个前提到底成立到什么程度。

## 方法

对 424 条 `fact` 全文扫描三类风险：

1. **遮蔽普及度**：多少条带有「某」字遮蔽标记。
2. **结构化 PII**：手机号（`1[3-9]` + 9 位）、身份证号（17 位 + 校验位）——出现即为明确泄漏，与人名政策无关。
3. **未遮蔽人名地标**：CAIL2018 的遮蔽针对**案件参与人**（被告/被害/证人 → 范某/胡某某/张某乙），但**不遮蔽用作地标的人名**（房东、店主）。扫描锚定"常见姓氏 + 1–2 非某汉字 + 场所名词（出租屋/养殖场/水产/饭店…）"这一模式，精确定位这类残留。

## 结果（2026-07-21，424 条）

| 检查项 | 结果 |
|---|---|
| 含「某」遮蔽标记 | **418 / 424（98%）** |
| 手机号 | **0** ✓ |
| 身份证号 | **0** ✓ |
| 未遮蔽人名地标候选 | 9 个 → 人工分类后**仅 1 个真实私人姓名** |

### 9 个候选的人工分类

**真实私人姓名（1 个）——需要决策：**

- **龚见兴**（`…陆任水产附近龚见兴出租屋宰杀…`）：一名**房东**的全名（姓龚+名见兴），仅因其出租屋靠近作案地点而作为地标出现，本身不是案件当事人。出现在 2 条判决（去重对，同一真实案件：广州花都杀猫案）。CAIL2018 遮蔽了案件参与人，但漏掉了这个纯地标性质的第三方姓名。

**商号名（2 个）——公开商业实体，低关切：**

- **陆任水产**（`…陆任水产附近…`）：水产店名，"陆任"读作人名亦可，但作为登记商号出现。
- **王老大水产品有限公司**（`…"王老大水产品有限公司"冷库…`）：登记公司名（"王老大"为俗称），公开商业实体。

**误报（5 个）——非人名：**

- **格林豪泰酒店**：全国连锁酒店品牌（GreenTree Inn），"林"是从"格林豪泰"中截出。
- **邓州超市**：邓州为河南地名（邓州市），"邓州超市"是地名商号。
- **江苏省海门市**："苏"从"江苏省"截出，实为地名+法院。
- **佑康棒冰批发部**："佑康"是棒冰品牌，"康"被截出。
- **金明宾馆**：宾馆品牌名，非私人全名。

## 决策（HRL-024，2026-07-21 DECIDED）

曾存在的张力：

- **支持保留（§8(b) 字面 + HRL-008 用户偏好）**：政策明写照抄文书；用户在 HRL-008 明确"细节越多越好，不要过滤"。
- **支持遮蔽（项目保护性默认 + HRL-008 唯一例外的同类逻辑）**：龚见兴是纯附带的第三方（房东，非当事人），其全名对研究零分析价值；遮蔽方向是"比信源更少具名"，与项目保护性默认一致。

**用户决定：保留原文，照 §8(b) 照抄 CAIL2018 文书遮蔽程度**（不采纳 AI 建议的冗余遮蔽）。龚见兴及 2 个商号名一律保持判决书原文——与 census 身份政策"照抄文书遮蔽程度、绝不跨文书拼接身份"一致，也与 HRL-008 一贯偏好一致。`judgments_census.csv` 无需改动。

## QA 结论

结构化 PII 零泄漏（0 手机号 / 0 身份证）、98% 参与人已被 CAIL2018 遮蔽、唯一未遮蔽私人姓名为一名附带房东，用户知情后确认可接受。**发布前隐私 QA 通过**（本轮 2026-07-21 数据快照）。
