# 收录与排除标准（Inclusion / Exclusion Criteria）

本文档摘录并展开自 `methodology.md` §2，聚焦"什么样的候选事件会成为 AHID-CN 的一条 Incident 记录"这一问题本身，供只关心收录边界、不需要通读全部方法论的读者使用。字段层面的定义以同目录下 `data_dictionary.csv` 为准。

## 地理范围

事件须满足以下三者之一：中国大陆境内发生并有公开报道；虽跨境发生但与中国大陆的机构、企业或个人活动直接相关；候选来源虽发布于海外平台，但内容本身声称事件发生在中国大陆。

## 事件类型

系统性收录以下类别：蓄意伤害行为（殴打、火烧、刺伤、投毒等）；持续折磨或致死；严重遗弃、饥饿或医疗忽视；以牟利或流量为目的的虐待拍摄；强迫动物争斗；非人道的捕捉、运输、收容或扑杀；机构性伤害，包括繁殖场、宠物店、动物园等场所内发生的伤害；涉及未成年人的动物暴力事件；引发政府调查或公共争议的事件。

## 默认不收录

以下类型默认排除，不进入候选池：不含具体事件的单纯观点表达；正常兽医治疗画面；普通"动物伤人"事件（即人被动物所伤，而非动物被伤）；无法溯源到任何具体来源的纯文字传言；仅有合成或伪造图片、却无现实事件主张支撑的内容；以人肉搜索个人信息为主要内容的帖子；一般性养宠纠纷且不涉及严重伤害。

## AHID-CN 语料的范围声明

本数据集的所有事件均以简体中文公开来源为主要信息来源语言，地理范围如上。未来若扩展其他语料（例如面向韩语来源的 AHID-KR），将独立定义其收录范围，不会影响本数据集已发布的记录。

## 本版本（v0.1）的采集方式：Tier D URL 驱动回填

本版本不做自动化平台发现或持续抓取，而是由项目负责人（PI）人工收集已知事件的公开 URL 线索，再由流水线执行归档、去重、归并、抽取与评分。这意味着本数据集的覆盖面直接受限于 PI 已知的事件线索范围，系统性小于未来自动发现阶段可能达到的覆盖面——这一点在 `known_limitations.md` 中作为"发现方式偏差"单独披露，此处仅作为收录方式本身的说明。

## 每条候选事件的人工把关

每一条进入本数据集的事件，在被采集前都经过 PI 本人的收录判断（不是仅由流水线自动纳入）；被判断不收录或判定为流水线内部测试用例的候选，不出现在本数据集中。这一逐条人工审核的过程记录在源代码仓库的 `docs/pipeline/candidate_incidents_seed.md` 与 `docs/human_review_log.md`（HRL-007、HRL-016、HRL-017）中，本数据集包不重复收录该审核记录的全文。
