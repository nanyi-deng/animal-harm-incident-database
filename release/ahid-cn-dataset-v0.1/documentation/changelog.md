# Changelog

格式参考 [Keep a Changelog](https://keepachangelog.com/) 的精神，但由于这是第一个版本，没有"变更"可比较，只记录这一版本包含什么、以及构建过程中值得留痕的节点。后续版本发布时，本文件会新增条目，说明相对上一版本增删了哪些事件、修复了哪些流水线问题、评分权重是否调整。

## [v0.1] — 尚未正式发布（本包为本地暂存构建）

### 数据集组成

30 条事件（AHID-CN-2026-0001 至对应 seed 编号，跳过标记为不收录或内部测试用例的候选），53 条来源，151 条 claim，34 条结构化机构回应记录。全部事件均通过 Tier D（URL 驱动回填）采集，逐条经项目负责人人工收录判断（`docs/human_review_log.md` HRL-007、HRL-016、HRL-017）。

### 流水线构建与关键修复节点

- Stage 0–5（种子录入 → 归档 → 去重与依赖分析 → claim 抽取 → 评分 → 导出）首次端到端跑通，2026-07-27。
- 归档阶段修复：未处理 gzip 压缩响应导致部分来源正文抓取失败；请求头过于精简触发内容分发网络的反爬校验，导致至少 1 条实际可访问的来源被误判为不可用（详见 `known_limitations.md` 与人工审计记录）。
- 依赖分析阶段修复：页面导航/页脚等模板文字稀释了正文相似度计算；一处来源正文由 JavaScript 动态渲染、抓取到的是加载占位符而非真实正文；"据媒体报道"一类泛化措辞被误判为共同引用信号。
- 评分阶段修复：反证/矛盾标记未正确传导至事件级别的 `disputed_flag`；四处评分查询错误地以"是否有抓取时间戳"而非"是否真正可访问"作为来源计入评分的条件，导致死链/被拦截来源仍计入评分。
- 新增 `responses_public` 表的抽取逻辑（此前该表未被任何环节写入）；发现并修复关键词匹配无法处理否定语义导致的分类反转问题；1 条谣言性质事件的 claim 因内容与"救助"类型不符被人工排除，理由见 `known_limitations.md`。

### 人工审计

第一轮 gold-standard 人工审计（2026-07-27，`human_review_log.md` HRL-018）：项目负责人核对全部 31 条流水线处理过的事件（含本数据集排除的 1 条内部测试用例），0 条内容错误，3 条轻微问题（均为来源可访问性问题，1 条经核实修复）。局限说明见 `known_limitations.md` 第三节。

### 尚未完成、阻塞正式发布的事项

许可证条款（HRL-005）、隐私 QA 签核（HRL-008）、Zenodo/ORCID 账号（HRL-009）均待定，详见主仓库 `docs/human_review_log.md`。
